[edit] Installation - WISE v 1.0.0.0

  1. Backup Config File: If reinstalling, create a backup of the config file in the ‘\OCS\Services\Service_Manager’ folder (filename "OCS_Service_Manager.config.XML")
  2. Install: Run SmartLink_SetUp.EXE
    1. Install Location: Select the folder that will store the config files
      • Use default of "C:\Program Files\OCS\Services\"
    2. Component Selection: Select either SmartLink Transfer (SLT) and/or SmartLink Service
    3. Settings Location: Select folder for trace logs, DBDefinition file, etc.
      • Use default of "C:\OCS\SmartLink\Services"
    4. Click next, wait for files to install, Click Finish
  3. Configure: Define the appropriate settings using service manager; my test settings are as follows:
    1. General tab: (The settings listed here are for testing; you'll need to coordinate with ProdOps and the customer for actual production settings.)
      • Email Server Location: "vm-smtp-c.ocs"
      • Send Email To: "mpierce@ocsys.com"
      • Unique Identifier: "mpierce" (or "Amsys", etc.)
      • Agency ID: "999999"
      • OCS Web Service URL: [use default]
        • I override for testing with: "http://10.0.3.80:8000/DataRetrievalWS.asmx"
        • Note: Double-click the text box to re-enable for edit (so can paste new test URL in place)
        • Note that if the test Web Service site is up and running, the circle will turn green...else will be red if site is down or url incorrect, or improperly configured
    2. SL Extractor tab:
      • Vendor Name: "Amedisys", "Cerner", "Misys", "McKesson", "OCS_Oasis"
      • Vendor Version: "1.0" (see comment on Cerner)
        • ...or appropriate version as can be derived from SmartLink Web Service DBDefinition file contents.
        • There are two versions for Cerner ("1.0" and "2.0") that correspond to the version of the Cerner database (Cerner 4.3 vs Cerner 5.0). To find out which version to use try the "TESTQUERY" from each VendorVersion in the DBDefinition file. The one that returns a row will be from the version you should use.
      • Database Connection String/Data Source Name:
        • Amedisys - "Provider=SQLOLEDB.1;Integrated Security=SSPI;Persist Security Info=False;Initial Catalog=mpierce_Amedisys_Source;Data Source=SQL64TEST"
          • This was a test connection to pull from a mocked up Amedisys MSSQL database (that contains 4 views)
        • Cerner - You must first create a data source, then enter its name into the 'Data Source Name' field. Note: When 'Cerner' is entered into the 'Vendor Name' field, the ellipsis button will disappear from beside the 'Data Source Name' field.
          • Create a Data Source:
            • Open the ODBC Data Source Administrator
            • 64bit OS: C:\Windows\SysWOW64\odbcad32.exe (IMPORTANT TO USE THIS 32bit VERSION OF THE TOOL)
            • 32bit OS: Control Panel->Administrative Tools->Data Sources (ODBC)
              • ODBC (tab)
                • Data Source Name = whateverYouWant
              • Login (tab)
                • Authentication = "Database"
                • User ID = "" (blank)
                • Password = "" (blank)
                • Action = "Start and connect to a database on this computer"
                • Note that you can use User ID = "viewonly" and Password = "noharm" to test the connection in ODBC, and to make a connection to view the data from Sybase...
                • Also, note that the free version of Sybase only supports a single connection, so if you are already connected, the SmartLinkClient will not be able to.
              • All other tabs
                • Use default values
        • McKesson, Misys
          • Database Connection string Click the ellipsis (...). The "Data Link Properties" form will open.
          • Select the "Provider" tab, scroll to the list's end and select "Microsoft OLE DB Provider for SQL Server" . Click the "Next" button.
          • On the "Connection" tab, enter the SQL Server name or select it from the drop-down list. Select the "Use Windows NT Integrated security:" radio button. Click the "Select the database" radio button, then select your McKesson vendor database from the drop-down list.
          • Click the "Test Connection" button, then the "OK" button.
      • Output Folder: [use default]
        • Default value is: "C:\OCS\SmartLink\Services\SLT\FileUpload\Upload"
      • Schedule Time: Change this as necessary for testing purposes
        • I typically set ahead 2 minutes then click save so that the service has time to auto-restart then process.
        • Note that you simply delete the file "OCS_DBExtractor_QueryStatus.XML" from the directory "C:\OCS\SmartLink\Services\OCS_DBX" to re-export the complete client database as if it were a first time install
    3. SL Transfer Tab:
      • Main Polling Path: [use default]
        • Default path is "C:\OCS\SmartLink\Services\SLT\FileUpload\Upload"
        • Typically leave Polling Path 1 and 2 blank unless client has a need to poll multiple paths
      • Go with default values on the remaining options
  4. Save and Exit: Save changes and exit, and don't start services yet when prompted.
  • When the "Save" button is clicked, the Smartlink client executes a vendor-specific query against the database. If the vendor name and database are mismatched (Example: Misys was entered as the Vendor Name and the database connection is pointing to a McKesson database, you'll see this error: 'Database did not pass validation. Please recheck database information and retry, or contact OCS Client Services.'
  • NOTE: For Cerner you cannot start the service at all using the "OCS Service Manager" because it will prevent the service from connecting to the Sybase database. This is only a problem w/ the non-enterprise version of Sybase that we test with. It will not be an issue for clients.
  1. Service Permissions:
    1. Go to Control Panel - Administrative Tools – Services
    2. Go to OCS_SmartLink_Extractor – right click – Properties – Log On Tab
    3. Add a user that has access to the database SmartLink will be using
      • I used me: "ocs\mpierce" plus my password to test; note the domain prefix "ocs\"
    4. Press OK to save changes
    5. Repeat for OCS_SmartLink_Transfer by using a new domain account with full access to the folders SLT will be using
    6. If needed, restore the the config files from the backup into the config folder
    7. Save changes and close dialog
    8. Re-start both services if they did not automatically do so

[edit] Installation - MSI v 2.0.0.2

  • Performing an over-the-top installation is not possible as v 1.0.0.0 used a WISE installer and v 2.0.0.x uses the Microsoft Installer (MSI).
  • Refer to the notes at Client Installation Instructions for instructions to save exisiting configuration files.
  • Uninstall the existing v 1.0.0.0 using the MS-Windows Control Panel's uninstall software option.
  • Install v 2.0.0.2 following the instructions at Client Installation Instructions

[edit] Troubleshooting

[edit] SmartLink Service Manager not completely removed

  • Error: 1001: While attempting to repair, uninstall, or install the SmartLink Service Manager you see this error:
    • 'Error 1001. An exception occurred whil uninstalling. This exception will be ignored and the uninstall will continue. However, the application might not be fully uninstalled after the uninstall is complete. --> The specified service does not exist as an installed service'
  • Situation: This can happen when the SmartLink Service Manager is open while the MSI uninstaller is run from within the MS-Windows Control Panel.
  • Solution: In testing, we ran a Microsoft tool, Windows Installer Clean Up. Microsoft no longer distributes this tool, because if carelessly used will create more problems.
    • If using the tool, be extremely careful to select only the '(All Users) Smartlink' installed product from within the displayed list and no others before clicking the 'Remove' button. This will remove SmartLink from the MSI database and allow you to install Smartlink.

[edit] OCS Service not uninstalled

  • Situation: You are unable to update an OCS service because the previous version was not uninstalled correctly. The service appears within the Services.msc console.
  • Solution: Remove the service from a command line then reinstall.
    • Start -> Run -> CMD.
    • Type SC Delete [ServiceNameHere] Example: SC Delete OCS SmartLink Extractor

[edit] SmartLink Client Side File Reprocessing

Steps to reprocess client files using the client's SmartLink Extractor service to extract files for resubmission to NRC.

  1. Go to the installation location for SmartLink.
    • Default location: C:\OCS\SmartLink\Services
  2. Edit the following file: SL_DBXQueryStatus.xml
  3. Set the StartDate and EndDate to the desired start time frame.
    • Example: “1/1/2015 00:00:00 AM”
    • All other settings can be left alone
    • You cannot set an actual end date. SmartLink will re-pull all information modified from the time you specify in the configuration file until the current date.
  4. Close SmartLink manager and it will process at the next scheduled time.
