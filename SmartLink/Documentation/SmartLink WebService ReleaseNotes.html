[edit] Deployment Guidelines

Production Location: \\SLENDER\BuildHost\Production_Releases\SmartLink_Server\WEB_SERVICE

[edit] Installation:

  1. Install & Configure MySQL if not already installed
    1. Installing MySQL:
      1. Location: Setup this location
      • Note: Can download from http://www.mysql.com/downloads/mysql/
      1. Run mysql setup exe ("mysql-essential-5.0.0-win32.exe" or equivalent); just follow defaults (note the uid="root" and the password you assign)
        • Note: If you get an error "Default storage engine innoDB is not available", then chances are you reinstalled without first uninstalling *and* deleting the old "ib_logfileN" files from "C:\Program Files\MySQL\MySQL Server 5.0\data" (the original transaction logs).
      2. Run mysql Connector Net msi next ("msql.data.msi" or equivalent); just follow defaults
      3. Run mysql GUI tools setup exe next ("mysql-gui-tools-5.0-r12-win32.msi" or equivalent); just follow defaults
    2. Configuring MySQL environment (databases/schemas, users, and tables added)
      1. Login to "MySQL Administrator"
        1. Run Start | All Programs | MySQL | MySQL Administrator
        2. Create a new Stored Connection (hit [...] button, name Connection with server name, username = "root", password = the password you assigned when configuring MySQL; use hostname = "localhost"; defaults & blank for the rest; click Apply & Close
        3. Select just saved Stored Connection from dropdown, localhost as server, Username = "root" and password= your selection; Click Ok
      2. Add Catalog "SmartLink"
        1. Select "Catalogs" from left pane
        2. Right-click in lower left pane and select "Create New Schema"
        3. Enter name "smartlink" for the catalog (schema/database)
      3. Add User "SmartLinkWebSvc"
        1. Select "User Administration" from left pane
        2. Click Add new user button
        3. User Information tab: MySQL User = "SmartLinkWebSvc"; password = whatever;
        4. Schema Privileges tab: Grant Select, Insert, Update, and Delete priveleges to user SmartLinkWebSvc
      4. Close "MySQL Administrator"
      5. Add Table "tblocswebservice"
        1. Run Start | All Programs | MySQL | MySQL Query Browser (login as username = root; pwd = you defined; and schema = smartlink
        2. Right-click in the ResultSet tab (main white pane) and click "Add New Script Tab"
        3. In the new script tab, run this SQL Script to Create Table (paste sql, click Execute button)
  2. Create Folder Structure where incoming files will be deposited, as well as a trace file for logging and set config file settings accordingly.
    • Subfolders: "ExcludedFiles", "Upload", "Working"
  3. Copy Web Service:
    1. From latest release folder: \\slender\BuildHost\Production_Releases\SmartLink_Server\WEB_SERVICE\
    2. To IIS website folder for this product (e.g.: E:\TESTSITES\SmartLinkWebSvc for my test machine)
  4. Update Web Account Permissions:
    1. From Windows Explorer, right-click To IIS website folder (e.g.: E:\TESTSITES\SmartLinkWebSvc for my test machine)
    2. Select Security tab
    3. Add Machine's "Internet Guest Account" and "Launch IIS Process Account" with Modify permissions (everything short of Full Control -- may be able to dial these settings down tighter, esp. in prod)
    4. Might consider replacing the above accounts with the group "WPG_IIS" instead (need to test first, no time now and the above settings work)
  5. Update DBDefinition File:
    1. Copy latest version of OCS_DBX_DBDefinition_Source.XML that is in folder: $\SmartLink_Server\DBDefinitionFile
    2. Paste into topmost directory of IIS website you just copied over
    3. Be sure to Share the DBDefinition file with read and write access "elibad" or person responsable to update the DBDefinition file (unless ProdOps will manage)
  6. Update Config File “web.config” with proper settings (in IIS Web Service parent directory)
    • UpLoadDir – Directory path to hold the uploaded files
    • WorkingDir – Directory path to hold the files being transmitted
    • ExcludedFilesDir – Directory path to hold the files with excluded file types
    • TraceFlag: This Flag defines whether the log should be used
    • TraceFileNamePath: File path of log file
    • DBDefinitionFileName: Location of DBDefinition source File
    • MySQLConnectionString: Connection string for MySQL database
      • My Test Connection String: "Server=localhost; Database=smartlink; Uid=SmartLinkWebSvc; Pwd=pass123; pooling=true;"
  7. Install Web Service via IIS Manager.
    1. Open IIS Manager, expand trees to Web Sites, right-click on "Web Sites" and select "New" then "Web Site"
    2. Enter "DataRetrievalWS" (or "SmartLinkWS") for the description and click next
    3. Enter or leave default the IP, Port Number, and/or Host Header (talk with IT to determine production settings here) and click next
    4. Browse to the Web Service path and click next (leave Allow anonymous access checked) and click next
    5. Check allow permissions for Read, Run Scripts, Execute, Write, and Browse (we will back off these permissions in production, but nice to have in dev so can see the web sesrvice self-assist / help page with method calls, etc.)
    6. Click Finish to close the wizard and create the site
    7. Right-click the new Web Site and click properties (in IIS Manager)
    8. Selest ASP.NET tab, select ASP.NET version dropdown to 2.0.xxxx (the default of 1.0.xxxx will error when you try to browse or link to the web service since the VB DLL was written using .NET framework 2.0 or higher)
    9. Select Documents tab, then click Add, and type in name "DataRetrievalWS.asmx". Then highlight the new filename and click Move Up until it is at the top of the list.
    10. Click Ok to save and close the two web site properties changes
    11. From IIS Manager, under the Web Sites tree and your new site, highlight the file "DataRetrievalWS.asmx", right-click and select Browse
      • Verify that the web service methods popup in a browser window indicating that the web service is up and running
